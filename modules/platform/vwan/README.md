# modules/platform/vwan

Deploys the **Azure Virtual WAN** — the backbone of the hub-and-spoke topology.
This is a **global** resource: deployed once (in `environments/_global`) and
shared across every region and environment.

## What it deploys

| Resource | Azure type |
|----------|-----------|
| Virtual WAN | `Microsoft.Network/virtualWans` |

## Naming & tags

The Virtual WAN name is generated by [`modules/naming`](../../naming) — there is
**no `name` input**; you pass the naming context (`subscription_id`,
`environment`, `description`, `unique_id`) instead. Tags are supplied as the
merged output of [`modules/tags`](../../tags).

`sku` defaults to **Standard**, which is required for secured hubs and routing
intent (used by Hub01).

## Usage

```hcl
module "vwan" {
  source = "../../modules/platform/vwan"

  resource_group_name = "rg-conn-global-prod"
  location            = "uksouth"
  sku                 = "Standard"

  subscription_id = "conn"
  environment     = "prod"
  description     = "vdi"
  unique_id       = "01"

  tags = module.tags.tags
}
```

## Inputs

| Name | Description | Type | Default |
|------|-------------|------|---------|
| `resource_group_name` | Target resource group | `string` | — |
| `location` | Azure region (home region for the global vWAN) | `string` | — |
| `sku` | `Basic` \| `Standard` | `string` | `Standard` |
| `tags` | Merged tag map (from `modules/tags`) | `map(string)` | `{}` |
| `subscription_id` | Short subscription code (naming) | `string` | — |
| `environment` | Environment segment (naming) | `string` | — |
| `description` | Short descriptor (naming) | `string` | `vdi` |
| `unique_id` | Uniqueness suffix (naming) | `string` | `""` |

## Outputs

| Name | Description |
|------|-------------|
| `vwan_id` | Virtual WAN resource ID — consumed by both hub modules |
| `vwan_name` | Generated Virtual WAN name |
